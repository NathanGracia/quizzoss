# Quizzoss

Quiz app that generates questions from Obsidian notes using Gemini AI. Built with FastAPI + vanilla JS.

## Features

- **Quiz sessions** вҖ” 10 questions drawn randomly from your notes, with score tracking
- **AI evaluation** вҖ” answers are evaluated by Gemini with three outcomes: RГ©ussi / Incomplet / ГүchouГ©
- **Inline chat** вҖ” after each answer, chat with a tutor who has full context (the note, the question, your answer, the evaluation)
- **Note library** вҖ” browse all notes as cards, grouped by topic
- **Note viewer** вҖ” read notes with rendered markdown, image support, and a sticky table of contents
- **Inline editor** вҖ” edit notes directly in the browser, protected by a password
- **Client-side routing** вҖ” notes have shareable URLs (`?note=path&section=heading`), browser back/forward works

## Local setup

**Requirements:** Python 3.11+, a Gemini API key.

```bash
git clone <repo>
cd quizzoss
pip install -r requirements.txt
cp .env.example .env   # then fill in the values
```

### Environment variables

Copy `.env.example` to `.env` and set the following:

| Variable | Required | Default | Description |
|---|---|---|---|
| `VAULT_ROOT` | вң“ | вҖ” | Absolute path to the root of your Obsidian vault |
| `NOTES_DIR` | | (vault root) | Sub-folder inside the vault to restrict scanning to |
| `GEMINI_API_KEY` | вң“ | вҖ” | Google AI API key |
| `GEMINI_MODEL` | | `gemini-2.5-flash` | Model used for chat |
| `GEMINI_EVAL_MODEL` | | `gemini-2.5-flash` | Model used for answer evaluation |
| `EDIT_PASSWORD` | | `""` (no auth) | Password required to save notes via the browser editor. Leave empty to disable protection (useful in dev). |

### Build the question bank

```bash
# 1. Scan the vault вҶ’ chunks.json
python main.py scan

# 2. Generate questions for each chunk вҶ’ questions.json
python main.py prebuild
```

`prebuild` is incremental: it skips chunks that already have a question. Use `--force` to regenerate everything.

### Run

```bash
python main.py serve
# вҶ’ http://localhost:8000
```

Or via uvicorn directly:

```bash
uvicorn api:app --reload
```

## Docker (production)

### Build and run

```bash
docker compose up --build -d
```

The app is available on port `8000`. The vault is mounted read-only; only `questions.json` is writable from the container.

> **Note:** `questions.json` must exist on the host before starting. Run `python main.py scan && python main.py prebuild` locally (or on the VPS) first.

### docker-compose.yml

The compose file expects two env vars from `.env`:

- `VAULT_ROOT` вҖ” path to the vault on the host (mounted into the container at the same path)
- All other variables are passed through `env_file: .env`

### Update workflow

```bash
git pull
docker compose up --build -d   # rebuilds the image, restarts the container

# If notes changed: re-scan and regenerate questions
python main.py scan
python main.py prebuild
# questions.json is mounted into the container вҖ” no restart needed
```

## VPS deployment with Syncthing

Recommended setup for keeping notes in sync between your local machine and the VPS:

1. Install Syncthing on both machines
2. Share your Obsidian vault folder (bidirectional or send-only from local)
3. Set `VAULT_ROOT` on the VPS to the Syncthing-managed folder path
4. Syncthing keeps the vault up to date in real time; `questions.json` is also mounted live, so running `prebuild` on the VPS applies immediately without restarting Docker

## CLI reference

```bash
python main.py scan               # extract chunks from vault
python main.py prebuild           # generate questions (incremental)
python main.py prebuild --force   # regenerate all questions
python main.py list               # list all chunks
python main.py list <filter>      # filter chunks by file or section name
python main.py show <n>           # show chunk #n in full
python main.py quiz               # run a quiz in the terminal
python main.py serve              # start the web server
python main.py watch              # watch vault for changes (auto re-scan)
```
