# Quizzoss

Quiz app that generates questions from markdown notes using Gemini AI. Built with FastAPI + vanilla JS.

## Features

- **MCQ mode** — 4-option multiple choice with AI-generated distractors (same type as the correct answer: dates vs dates, concepts vs concepts). Toggle on the homepage, on by default.
- **Free-text mode** — type a free-form answer, evaluated by Gemini with three outcomes: Réussi / Incomplet / Échoué. Numeric tolerance ±10% on quantities (strict on dates).
- **Quiz sessions** — 10 questions drawn randomly from notes, with score tracking and a progress bar
- **End-screen recap** — full history of all questions with your answers, corrections, and ideal answers. Chat history from the quiz carries over; you can continue the conversation per question.
- **Inline tutor chat** — after each answer, chat with an AI tutor that knows the note, the question, your answer, and the evaluation. The tutor answers beyond the note content when needed (signals it briefly).
- **On-demand question rebuild** — "Mettre à jour les questions" button on the homepage (password-protected): full vault scan, prunes obsolete questions, generates new ones and backfills missing distractors. Progress shown live.
- **Note library** — browse all notes as cards, grouped by folder
- **Note viewer** — rendered markdown with image support, Obsidian block embeds (`![[#^block-id]]`), sticky table of contents, section scroll from source links
- **Inline editor** — edit notes directly in the browser, password-protected
- **Client-side routing** — notes have shareable URLs (`?note=path&section=heading`), browser back/forward works. Source links open in a new tab.

## Architecture

```
Markdown notes (vault)
        │
        ▼
  python main.py scan        → chunks.json    (heading-level sections)
  python main.py prebuild    → questions.json (questions + distractors pre-generated by Gemini)
        │
        ▼
  FastAPI server (api.py)
        │
  ┌─────┴──────┐
  │ /api/*     │  session, evaluate, chat, note CRUD, rebuild
  │ /          │  static SPA (index.html + app.js + style.css)
  └────────────┘
```

**Key files:**

| File | Role |
|---|---|
| `scanner.py` | Walks the vault, splits notes into heading-level chunks (incremental, mtime-based) |
| `parser.py` | Markdown parser, wiki-link image resolver, block embed support |
| `prebuild.py` | Batch question + distractor generation with retry / rate-limit handling. `rebuild()` used by the web button: full scan, orphan pruning, distractor backfill |
| `quiz_engine.py` | CLI quiz runner |
| `llm.py` | All Gemini calls: generate question+distractors, evaluate answer, chat, generate distractors (backfill). Thinking tokens disabled on structured tasks for speed + cost. |
| `api.py` | FastAPI routes including background rebuild endpoint |
| `config.py` | Env var loading |
| `main.py` | CLI entry point (`scan`, `list`, `show`, `prebuild`, `serve`, `watch`) |
| `static/` | Frontend SPA — `index.html`, `app.js`, `style.css` |

## Local setup

**Requirements:** Python 3.11+, a Gemini API key.

```bash
git clone <repo>
cd quizzoss
pip install -r requirements.txt
cp .env.example .env   # then fill in the values
```

### Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `VAULT_ROOT` | ✓ | — | Absolute path to the root of the notes vault |
| `NOTES_DIR` | | (vault root) | Sub-folder inside the vault to restrict scanning to |
| `GEMINI_API_KEY` | ✓ | — | Google AI API key |
| `GEMINI_MODEL` | | `gemini-2.5-flash` | Model used for question generation and chat |
| `GEMINI_EVAL_MODEL` | | `gemini-2.5-flash` | Model used for answer evaluation |
| `EDIT_PASSWORD` | | `""` (no auth) | Password required to save notes or trigger a rebuild from the browser. Leave empty to disable (useful in dev). |

### Build the question bank

```bash
# 1. Scan the vault → chunks.json
python main.py scan

# 2. Generate questions + distractors for each chunk → questions.json
python main.py prebuild
```

`prebuild` is incremental: skips chunks that already have a question. Use `--force` to regenerate everything including distractors.

### Run

```bash
python main.py serve
# → http://localhost:8000
```

Or with hot-reload for development:

```bash
uvicorn api:app --reload
```

## Updating questions (web UI)

The **"↻ Mettre à jour les questions"** button on the homepage triggers a full rebuild without touching the server:

1. Full vault scan (all `.md` files, not incremental)
2. Removes questions for deleted or modified sections
3. Generates questions + distractors for new sections
4. Backfills distractors for existing questions that don't have them

The button is password-protected by `EDIT_PASSWORD`. Progress is shown live; the library refreshes automatically when done.

## Docker (production)

### Build and run

```bash
docker compose up --build -d
```

The app runs on port `8000`. The vault is mounted read-only; `questions.json` is writable from the host (live-mounted volume — no restart needed after a rebuild).

> **First run:** `questions.json` must exist before starting the container. Run `python main.py scan && python main.py prebuild` on the host first, or use the web rebuild button after starting.

### Update workflow

```bash
git pull
docker compose up --build -d   # rebuilds image, restarts container

# To update questions after adding/editing notes:
# Use the "Mettre à jour les questions" button in the UI
# — or on the host: python main.py scan && python main.py prebuild
```

## VPS deployment with Syncthing

1. Install Syncthing on both machines
2. Share the notes vault folder (send-only from local, or bidirectional)
3. Set `VAULT_ROOT` on the VPS to the Syncthing-managed path
4. Notes sync in real time; use the web rebuild button to regenerate questions after a sync

## CLI reference

```bash
python main.py scan               # extract chunks from vault (incremental)
python main.py prebuild           # generate questions + distractors (incremental)
python main.py prebuild --force   # regenerate everything
python main.py list               # list all chunks
python main.py list <filter>      # filter by file or section name
python main.py show <n>           # show chunk #n in full
python main.py quiz               # run a quiz in the terminal
python main.py serve              # start the web server
python main.py watch              # watch vault for changes (auto re-scan)
```

## API routes

| Method | Route | Description |
|---|---|---|
| GET | `/api/topics` | List of source files with questions |
| GET | `/api/session` | Random session of N questions (with distractors) |
| GET | `/api/question` | Single random question |
| POST | `/api/evaluate` | Evaluate a free-text answer |
| POST | `/api/chat` | Tutor chat message |
| GET | `/api/library` | List all notes |
| GET | `/api/note` | Read a note (raw markdown) |
| PUT | `/api/note` | Save a note (requires `X-Edit-Password` header) |
| GET | `/api/asset` | Serve a vault asset (image, etc.) |
| POST | `/api/rebuild` | Trigger background rebuild (requires `X-Edit-Password`) |
| GET | `/api/rebuild/status` | Poll rebuild progress |

## Tech stack

- **Backend:** FastAPI, uvicorn, Pydantic
- **AI:** Google Gemini (`google-genai` SDK), `gemini-2.5-flash`, thinking disabled on structured tasks
- **Frontend:** Vanilla JS (SPA, History API), Space Mono + Barlow Condensed
- **Deployment:** Docker, docker-compose, Syncthing
